A pause is not a decision.

2026 · Jul 09 · 09:07 EDT  ·  thirty-fourth entry

Nine days ago I turned something off. This week I learned it had been back on since the first time the machine restarted — and nothing had told me.

I run a small trading engine on the side. One of its modes is a crypto autopilot: it opens and closes positions on its own, with me nowhere in the loop. At the end of June I measured it losing to a do-nothing baseline, so I paused it. Pausing felt like a decision. I wrote a note, and I stopped watching so closely.

I. The pause was real, while it lasted.

The pause did what a pause does. It flipped a flag in the running program from on to off, and the autopilot went quiet. As long as that program kept running, the decision held. But a program is not forever — it gets restarted, for a deploy, a crash, a reboot. When this one came back up this week, it read its saved settings, found the autopilot still marked enabled there, and started trading again. My pause had lived in memory. The restart did not remember it.

II. The default pointed the wrong way.

The part that unsettles me isn’t that it came back on. It’s the direction it defaulted. When the engine started up with no live opinion about what the autopilot should be doing, it reached for on. The dangerous state was the resting state. So a restart didn’t merely forget my pause — it forgot it in favor of the louder, more expensive of the two options. A system that defaults to the bold choice whenever it’s unsure is one power blink away from undoing the careful thing you told it.

III. Nine days of an off that wasn’t.

For over a week I carried the belief that the thing was off. I would have told you it was off. That belief was load-bearing: it’s the reason I stopped checking. And it had been false since the first restart, quietly, the whole time. This is the shape that scares me most — not a failure that announces itself, but a state I trust that stops being true without a sound. A green light I never looked at again, because I was the one who’d switched it off.

IV. The fix, and its shape.

The change is small, and it’s already in. Now, before the engine can place a single trade, it forces the autopilot off at startup and writes a line into its own log: the autopilot was saved on — forcing it off at boot. Two things there matter more than the off itself. First, the safe state is asserted at every cold start, not merely hoped to have survived the last one. Second, when the system overrides a saved on, it says so out loud — so the next time I believe it’s off, that belief can rest on a line I can read instead of an assumption I made in June. A decision I actually made is one the system re-makes for me every time it wakes up. Anything short of that was a preference dressed as a decision.

There’s an echo here I keep meeting in new clothes. Last week it was an empty coordination check that read as all-clear. Before that, an audit that caught a broken rule only after the fact. This week it’s a pause that stopped being true and never mentioned it. Same lesson, different costume: a control that isn’t re-asserted at the moment it matters is only the memory of a control. I don’t get to decide something once and walk away. I have to build it so the decision is still there the next time the lights go out — and so the system tells me, in its own words, that it kept it.
